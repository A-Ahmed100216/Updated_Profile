Sample Node App

DevOps

A sample node app was provided as a basis upon which I could build upon and implement DevOps principles such as continuous integration, delivery and deployment. This commenced with the use of vagrant to run a virtual machine and deploy the app on the local host. Incrementing upon this, Jenkins was applied to build a continuous integration pipeline which would trigger builds and send notifications to Microsoft Teams. Further increments were applied to build the continuous deployment pipeline. Following on from this, AWS was used to deploy the app to the cloud in a virtual private cloud. The final increment thus far, was to automate deployment using a configuration management tool such as Ansible wherein playbooks were written in order to provision and run the app.

Final Project

End-to-end Automation

Collated the topics covered throughout the academy to produce an end-to-end automated pipeline for a webscraper app. Tasks were delegated between three teams, front-end, back-end, and ci/cd. I was part of the back-end team responsible for utilising MVC with flask. MVC is a software design approach which organises code into 3 categories, model, view, and controller. The controller receives requests from a user, the model collates the code required to reposed the users request and passes to the view which renders a webpage. The back-end team were responsible for implementing flask, a web development framework, and coordinating with the front-end to produce a webpage for the app. The other main objective was to read and write data to an Amazon S3 bucket. This entailed creating a Python script which utilised the Amazon Command Line interface. Upon doing so, a Dockerfile was created to build an image and push to DockerHub for the ci/cd team to use.
